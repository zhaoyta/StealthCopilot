# 架构说明

::: info 面向开发者
本页面包含技术细节，供贡献者和技术用户参考。普通用户请参阅[快速开始](./guide/)。
:::

StealthCopilot 基于三条核心数据管道构建，详细设计文档见仓库 `openspec/specs/prd.md`。

## 三条核心管道

### 听力链（目标延迟 ≤500ms）

```
会议音频 → 虚拟声卡捕获
  → 讯飞 RTASR 实时语音转写（WebSocket，只启用转写）
      ├─ 原文 → 讯飞机器翻译文本接口 → 译文 → 幽灵提词窗字幕
      ├─ 译文 → 系统语音播报 → 面试者本地监听输出
      └─ 原文 → 意图识别 → RAG 检索 → DeepSeek 回答生成 → 幽灵提词窗建议
```

耳机译文播报必须和会议虚拟麦克风隔离：它只进入用户本地监听输出，不进入 Zoom/Teams 的麦克风输入。当前系统语音后端使用 OS 默认输出，生产使用时应将默认输出切到耳机。

### 说话链（目标延迟 ≤1.2s）

```
物理麦克风 → 本地 VAD 检测说话结束
  → 讯飞同声传译（源语言语音 → 源语言文本 + 目标语言文本）
  → 讯飞机器翻译文本接口兜底 / DeepSeek 润色
  → 默认音色或讯飞声音复刻流式 TTS
  → 虚拟麦克风输出
```

等待期间向虚拟麦克风写入静音数据，防止中文语音泄露给面试官。

### 数字人视频输出（说话链可选输出模式）

```
说话链 TTS PCM
  → Simli AI WebRTC 视频
  → ffmpeg VP8 解码
  → 本机 OBS Browser Source: http://127.0.0.1:18765/
  → OBS Virtual Camera
  → 会议软件摄像头
```

Simli 当前只提供视频同步。会议听到的声音仍来自本地 TTS 写入虚拟麦克风；为补偿 Simli、ffmpeg、OBS 和会议软件的累积视频延迟，Simli 模式会默认延迟约 700ms 写入本地虚拟麦克风。

StealthCopilot 不注册自研虚拟摄像头驱动。OBS App 必须运行，OBS 场景必须包含应用提供的浏览器源，并由 OBS Virtual Camera 暴露给会议软件。

## 熔断保护

当云端服务（讯飞语音/讯飞声音复刻/Simli）出现连接问题时，系统停止当前失败输出并在前端提示。用户可关闭数字人输出，保留说话链虚拟麦克风音频继续会议。
