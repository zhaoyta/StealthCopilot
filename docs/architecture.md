StealthCopilot 基于三条核心数据管道构建，详细设计文档见仓库 `openspec/specs/prd.md`。

## 三条核心管道

### 听力链（目标延迟 ≤500ms）

```
会议音频 → 虚拟声卡捕获
  → 讯飞实时语音翻译（单次 WebSocket 调用）
      ├─ 译文 → 幽灵提词窗字幕
      ├─ 译文 → 系统语音播报 → 面试者本地监听输出
      └─ 原文 → 意图识别 → RAG 检索 → DeepSeek 回答生成 → 幽灵提词窗建议
```

耳机译文播报必须和会议虚拟麦克风隔离：它只进入用户本地监听输出，不进入 Zoom/Teams 的麦克风输入。当前系统语音后端使用 OS 默认输出，生产使用时应将默认输出切到耳机。

### 说话链（目标延迟 ≤1.2s）

```
物理麦克风 → 本地 VAD 检测说话结束
  → 讯飞语音翻译（中文语音 → 英文文本）
  → ElevenLabs 流式 TTS（克隆声音）
  → 虚拟麦克风输出
```

等待期间向虚拟麦克风写入静音数据，防止中文语音泄露给面试官。

### 视频链（目标 ≥30fps，音视频同步误差 ≤40ms）

```
物理摄像头 → Simli AI 口型同步处理
  → A/V 环形缓冲区（时间戳对齐）
  → 虚拟摄像头输出
```

## 熔断保护

当云端服务（讯飞/ElevenLabs/Simli）出现连接问题时，系统将在 10ms 内自动切换到本地直通模式：真实麦克风和摄像头直接输出，面试不中断。
