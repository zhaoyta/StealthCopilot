## Context

当前实现已经具备讯飞声音复刻的主要后端入口：

- `GetXunfeiVoiceTrainText`：读取讯飞训练文本，供前端展示朗读。
- `CloneVoice`：提交 WAV 训练音频，保存返回的 Task ID。
- `QueryXunfeiVoiceCloneStatus`：查询训练任务，成功时保存 Asset ID。
- `XunfeiVoiceCloneProvider`：说话链运行时使用 Asset ID 进行 WebSocket 流式合成。

需要重新梳理的是产品状态流，而不是再引入一个新的 TTS 平台。

## Flow

### 1. 凭证准备

用户在 Setup Step 3 或设置面板中只保存讯飞声音复刻 AppID、API Key、API Secret。连接测试只验证训练接口可用；没有 Asset ID 时也可以返回“凭证可用，尚未完成训练”。Task ID 和 Asset ID 由声音复刻流程写入系统存储，不允许用户在密钥配置中手填。

### 2. 训练文本获取

Step 4 打开时后端向讯飞请求训练文本。请求失败时前端可以展示内置兜底文案，但提交训练时后端仍以讯飞接口返回的训练文本 ID 和文本段 ID 为准。

### 3. 录音与提交

用户按训练文本录制 WAV。提交时后端创建训练任务，上传音频和文本段信息，保存 Task ID。此时状态应是 `submitted`，不是 `done`。

### 4. 训练状态查询

用户可手动查询，也可以后续补自动轮询。查询成功并返回 Asset ID 后保存到 Keychain，状态才进入 `done`。训练中保持 `submitted`；失败时显示失败原因并允许重新录制提交。

### 5. 说话链使用

说话链启动前必须选择 TTS 音色模式。默认音色模式无需 Asset ID，可以直接把目标语言文本合成为默认人声音频并写入虚拟麦克风。个人复刻音色模式需要检查 AppID、API Key、API Secret、Asset ID；配置完整时，目标语言文本通过讯飞声音复刻 WebSocket 合成 24k raw audio chunk，再写入虚拟麦克风。

## Decisions

### D1: 只保留讯飞声音复刻作为默认个人音色方案

当前项目的听力链、机器翻译和说话链已围绕讯飞服务组织。声音复刻继续使用同一供应商可以减少用户凭证数量，并让连接测试、帮助文档和故障提示保持一致。

### D2: Task ID 和 Asset ID 是两个不同状态

Task ID 表示训练已提交，Asset ID 表示可用于 TTS 输出。它们是系统与讯飞接口交互产生的流程状态，UI 可转译为“训练中/已完成”，但不应把原始 ID 当成用户需要配置的密钥字段。

### D3: 默认音色是正式可用路径，不是错误降级

用户可以主动不复刻声音，系统使用默认音色继续完成说话链。没有 Asset ID 时，UI 和日志必须明确显示“默认音色”，不得让用户误以为面试官会听到个人复刻音色。

## Risks

- 训练接口耗时或状态码语义可能变化，需要保留清晰错误消息。
- 浏览器录音采样率与讯飞训练质量要求可能不完全匹配，后续可能需要后端重采样或质量检测。
- 已提交任务在重启后需要通过保存的 Task ID 恢复查询入口。
